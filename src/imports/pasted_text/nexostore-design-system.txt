Actúa como un Product Designer Senior, UX/UI Designer, especialista en diseño de e-commerce, sistemas de diseño, accesibilidad, responsive design y handoff para equipos de desarrollo.

Debes crear en Figma el diseño completo, moderno, accesible y profesional de un e-commerce llamado “Urban Sport Store”.

No generes una explicación general ni una sola pantalla aislada. Diseña un archivo Figma organizado, con sistema de diseño, componentes reutilizables, prototipos navegables, variantes, diseño responsive y documentación de handoff para un equipo de desarrollo React + TypeScript.

El producto es una tienda online moderna de tecnología, accesorios y dispositivos inteligentes. La arquitectura debe permitir que el mismo diseño se adapte después a otros nichos como moda, hogar, belleza o deportes.

## Objetivo visual

Diseña una experiencia:

- Moderna.
- Limpia.
- Confiable.
- Premium pero accesible.
- Rápida de comprender.
- Orientada a conversión.
- Mobile first.
- Accesible.
- Preparada para e-commerce real.
- Adecuada para clientes y administradores.

El diseño debe equilibrar una estética tecnológica elegante con claridad comercial. Evita interfaces recargadas, gradientes excesivos, sombras pesadas, bloques enormes de texto, botones ambiguos y elementos decorativos que no aporten a la compra.

## Sistema visual

Crea un sistema de diseño completo.

### Colores

Crea tokens con nombres semánticos, no solo colores físicos.

Propuesta de estilo:

- Primario: azul eléctrico o azul profundo moderno para CTAs.
- Secundario: violeta o cyan sutil para estados complementarios.
- Neutrales: escala de grises fría, clara y profesional.
- Fondo principal claro.
- Superficies blancas.
- Texto principal oscuro y legible.
- Texto secundario con contraste adecuado.
- Colores semánticos: éxito, error, advertencia, información.
- Estados: hover, focus, pressed, disabled.
- Colores de descuento y stock bajo sin depender únicamente del color para transmitir significado.

Crea escalas de color y tokens como:

- `color/brand/primary`
- `color/surface/default`
- `color/text/primary`
- `color/text/secondary`
- `color/border/default`
- `color/feedback/success`
- `color/feedback/error`
- `color/feedback/warning`
- `color/action/disabled`

Garantiza contraste visual suficiente para texto, controles y estados.

### Tipografía

Usa una combinación moderna y legible, por ejemplo:

- Inter, Manrope, DM Sans o una fuente sans-serif equivalente.

Define estilos tipográficos:

- Display.
- H1.
- H2.
- H3.
- H4.
- Body large.
- Body.
- Body small.
- Label.
- Caption.
- Overline.
- Price large.
- Price regular.

Define pesos, tamaños, line height y espaciados consistentes. Mantén legibilidad en dispositivos móviles.

### Espaciado y layout

Crea una escala de espaciado de 4 u 8 puntos:

- 4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80, 96.

Define:

- Grid desktop de 12 columnas.
- Grid tablet de 8 columnas.
- Grid mobile de 4 columnas.
- Márgenes y gutters consistentes.
- Máximo ancho de contenido para escritorio.
- Breakpoints para mobile, tablet, laptop y desktop.

### Forma y elevación

Define:

- Radios: pequeño, medio, grande y circular.
- Bordes.
- Sombras: sutil, media y elevada.
- Estados de foco visibles.
- Separadores.
- Superficies para cards, modales, drawers y tablas.

## Prototipo navegable

Crea conexiones de prototipo para estos flujos:

### Flujo 1: Compra como visitante

Inicio -> Catálogo -> Detalle de producto -> Agregar al carrito -> Carrito -> Login/Registro -> Checkout -> Confirmación -> Detalle de pedido.

### Flujo 2: Compra como cliente

Inicio -> Búsqueda -> Detalle -> Carrito -> Checkout -> Seleccionar dirección -> Envío -> Confirmación -> Historial de pedidos.

### Flujo 3: Recuperación de contraseña

Login -> Olvidé mi contraseña -> Confirmación de correo -> Restablecer contraseña -> Login.

### Flujo 4: Administración de producto

Login admin -> Dashboard -> Productos -> Crear producto -> Validación de formulario -> Guardar -> Tabla de productos -> Editar producto.

### Flujo 5: Gestión de pedidos

Dashboard admin -> Pedidos -> Detalle -> Cambiar estado -> Confirmación -> Estado actualizado.
